# Vision Inference

Monorepo for a separated object-detection frontend and backend.

```text
.
├── backend/
│   ├── src/inference/
│   └── tests/
├── frontend/
│   └── app/
├── docker-compose.yml
└── README.md
```

The backend is a FastAPI service using Ports and Adapters / Onion Architecture.
The frontend is a Next.js application that uploads media to the backend and
renders image and sampled-video detection results.

## Architecture

- `backend/src/inference/domain/` contains immutable value objects and validation
  rules. It imports only standard-library code.
- `backend/src/inference/service_layer/` owns use cases and core-owned ports:
  `AbstractInferenceEngine` and `AbstractMediaProcessor`.
- `backend/src/inference/adapters/` contains concrete infrastructure for YOLO
  inference and OpenCV media processing.
- `backend/src/inference/entrypoints/` contains FastAPI routes, DTOs, CORS, and
  exception-to-HTTP mapping.
- `frontend/` is a separate browser adapter. It calls `POST /v1/detect/upload`
  through `NEXT_PUBLIC_API_BASE_URL`.

## Docker

Place YOLO weights at `models/yolov8n.pt`, then run:

```bash
docker compose up --build
```

Open:

- Frontend: `http://localhost:3000`
- Backend health check: `http://localhost:8000/health`
- Backend API docs: `http://localhost:8000/docs`

The compose file mounts `./models` into the backend container and sets:

```text
MODEL_PATH=/models/yolov8n.pt
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

## Local Development

Install backend dependencies:

```bash
cd backend
uv sync --extra dev --extra vision
```

For fast backend tests without the real model stack, the vision extra is
optional:

```bash
cd backend
uv sync --extra dev
```

Install frontend dependencies:

```bash
cd frontend
bun install
```

Run both apps in separate terminals from the repository root:

```bash
make backend
make frontend
```

The root `make backend` target points `MODEL_PATH` at `../models/yolov8n.pt`
after changing into `backend/`.

## API

Upload image or video media:

```bash
curl -X POST http://localhost:8000/v1/detect/upload \
  -F "file=@sample.jpg;type=image/jpeg"
```

Supported image types are JPEG and PNG. Supported video types are MP4,
MOV/QuickTime, AVI, and WebM. Videos are sampled at
`VIDEO_SAMPLE_INTERVAL_SECONDS` and capped by `MAX_VIDEO_FRAMES`.

Image responses are shaped as:

```json
{
  "kind": "image",
  "result": {
    "model": "yolov8n",
    "image": { "width": 640, "height": 480 },
    "inference_ms": 12.5,
    "detections": [],
    "created_at": "2026-05-28T00:00:00Z"
  }
}
```

Video responses contain sampled frame results:

```json
{
  "kind": "video",
  "sample_interval_seconds": 1.0,
  "frames": [
    {
      "model": "yolov8n",
      "image": { "width": 640, "height": 480 },
      "inference_ms": 12.5,
      "detections": [],
      "created_at": "2026-05-28T00:00:00Z",
      "frame_index": 0,
      "timestamp_seconds": 0.0
    }
  ]
}
```

## Checks

```bash
make test
make lint-imports
make typecheck
make frontend-typecheck
make frontend-build
```

The backend test suite uses fake `AbstractInferenceEngine` and
`AbstractMediaProcessor` implementations, so it is fast and deterministic.

## Demo Assets

Bundled demo clips live in `frontend/public/assets/demo/`. They are synthetic
assets generated by `backend/scripts/generate_demo_videos.py`, so the repository
can show sample videos without redistributing third-party traffic-camera
footage.
